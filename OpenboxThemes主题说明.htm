主题安装路径

Openbox 主题所在目录：

  • 系统内置主题位于：/usr/share/themes。
  • 用户自定义主题位于 ~/.local/share/themes 或者 ~/.themes。

主题选择与切换

ObConf 这个小程序能简单直观的选择、切换主题。另外还有一些 Pipe menus 也能做到这一点。

想要自己动手修改配置文件的话，去看看 rc.xml 这个文件吧。参考 Configuration guide。

主题文件结构 

主题目录 (如 /usr/share/themes)
 |
 +-> ThemeName  (主题名称，如 Clearlooks)
      |
      +-> openbox-3  (Openbox 3 格式)
           |
           |-> themerc  (主题核心文件，也是此文档的主角)
           |
           |-> max.xbm  (标题栏按钮图案)
           |-> close.xbm
           ...
           +-> shade.xbm

ObConf 可以将它们打包成一个 .obt 文件。

数据类型

整数型

就是普通的数字。如：

window.handle.width: 3

文本对齐方式

可选项有：Left, Center , Right 。如：

menu.title.text.justify: Left

纹理

Textures 选项指定了元素的基础视觉样式。这是最丰富多彩的一个配置项，要搞清楚它有一点小难度。

Textures 可以是多个选项值的组合，各值不分先后顺序。下面的 | 表示 "或者"，[] 表示可选可不选。

parentrelative | ((solid | gradient gradient-type) [border] [interlaced])

下面逐个说明。

Parentrelative

Parentrelative 基本就是继承，不过有些元素不支持它，后面相应的地方会有说明。

例子：

window.active.label.bg: Parentrelative
window.inactive.label.bg: Parentrelative Raised

均匀填充

以单一颜色均匀平铺填充，如：

menu.items.bg:       Solid Flat
menu.items.bg.color: #f8f5f2

渐变填充

要使用 gradient 必须同时指定渐变类型，以及至少两个颜色值：color 和 colorTo。

可选的渐变类型有：

  • Diagonal - 左上至右下
  • CrossDiagonal - 右上至左下
  • Pyramid - 径向渐变，周围至中心
  • Horizontal - 水平渐变，从左至右
  • MirrorHorizontal - 水平镜像渐变，从左至中再镜像至右
  • Vertical - 垂直渐变，从上至下
  • SplitVertical - 双向垂直渐变，从中间往上并同时往下

如：

menu.title.bg:         Gradient Vertical Raised 
menu.title.bg.color:   #658fb5
menu.title.bg.colorTo: #4d6982

双向垂直渐变

SplitVertical 有两个渐变方向，所以需要多加两个颜色值：color.splitTo 和 colorTo.splitTo，用作渐变到顶部和底部的高亮颜色。如果不指定，则会按这个公式自动取值：color * 5/4, colorTo * 17/16 。

例子：

menu.title.bg:                 Gradient SplitVertical Raised 
menu.title.bg.color:           #658fb5
menu.title.bg.color.splitTo:   #7595b9
menu.title.bg.colorTo:         #4d6982
menu.title.bg.colorTo.splitTo: #557485

边框

采用均匀填充和渐变填充的元素，可以设置边框属性。边框类型有：Flat(扁平), Raised(凸起), Sunken(凹陷)。如果不配置框属性，元素将表现为 Raised(凸起) 效果。

如果指定边框类型为 Flat(扁平) 型，则必须同时指定边框颜色，否则将看不到边框效果。

例子：

window.active.button.unpressed.bg:              Gradient Vertical Flat Border
window.active.button.unpressed.bg.border.color: #3d4c5a

Raised(凸起) 和 Sunken(凹陷) 的边缘有两种类型，默认是外边缘，指定 Bevel2 则表示内边缘。这可以用来表现按下、弹起状态。

凹凸光影的强度也是可配置的：

highlight 值指定高亮区，取 0 值则没有高亮效果，取 128(默认值) 表示 50% 亮度，取 256 表示 100% 亮度，512 则为 200% 亮度，依此类推。

shadow 值指定阴影区，取值范围：0 至 256，取 0 值则没有阴影效果，值越大颜色越暗，取 256 则全黑。默认值为 64 (大约 25% 亮度)。

例子：

window.inactive.button.disabled.bg:           Gradient Diagonal Raised
window.inactive.button.disabled.bg.color:     rgb:50/54/58
window.inactive.button.disabled.bg.colorTo:   black
window.inactive.button.disabled.bg.highlight: 128
window.inactive.button.disabled.bg.shadow:    64

window.inactive.button.toggled.pressed.bg:          Gradient Diagonal Raised Bevel2
window.inactive.button.toggled.pressed.bg.color:    rgb:50/54/58
window.inactive.button.toggled.pressed.bg.colorTo:  black

隔行扫描

就是用两种颜色交替的画横线，所以指定 interlaced 的同时还需要指定扫描线的颜色。如：

window.inactive.title.bg: Solid Flat Interlaced
window.inactive.title.bg.color: #f5f5f5
window.inactive.title.bg.interlace.color: #f6f6f6

颜色

颜色就是 color，除了 RGB 值，还支持 "颜色名称"。

颜色名称

Wikipedia 有一份 X11 的颜色名称清单，这里更详细。

例子：

menu.items.active.text.color: white
window.active.grip.bg.color: grey40

RGB 值

RGB 值有两种写法：#rrggbb 和 rgb:rr/bb/gg。不如 "茴" 字的写法多。但要命的是 #fff 表示 #f0f0f0 而不是 #ffffff。

例子：

window.active.grip.bg.color: #658fb5
window.active.label.text.color: #fff
menu.items.active.bg.color: rgb:90/94/98
window.active.title.bg.color: rgb:6/9/c

文字阴影描述法

文字的阴影效果用一段特别的文字来描述。其中包括：shadow, shadowtint, shadowoffset。

文字默认没有阴影效果，设置 shadow=y 才能启用。

shadowtint 指定阴影的颜色和透明度，默认为 50% 透明的黑色。取值范围是：-100 至 100。取 0 值表示完全透明，取 100 表示完全不透明，其中正数表示黑色，负数表示白色。

shadowoffset 指定阴影与原文字的距离，默认为 1，因为阴影没有模糊效果，所以设置为 0 时阴影不可见。

再说一点，描述文字阴影的文字，要写在文字的 font 属性下。如：

window.active.label.text.font:shadow=y:shadowtint=70:shadowoffset=1

元素对象

就是构成窗口或菜单的各种元素，这当然就是重点啦。后面的说明会采用以下这样的格式

类型 整数型
默认值 1
取值范围 0-100
Parentrelative 是否支持 Parentrelative 继承

类型，就是属性取值的类型

默认值，就是不给属性明确指定值时，该属性会自动选取的值。

取值范围，就是允许取值的范围。

继承，为元素指定此属性，可以让它继承其上一级的样式，前提是此属性对该元素有效。

按钮装饰图案

标题栏按钮图案、子菜单标记图案，是一种 1-bit xbm (X Bitmaps) 图案文件。

这些图案文件，请存放在主题文件 themerc 所在的目录，就如 主题文件结构 说明的那样。

系统内置的图案文件位于 /usr/share/doc/openbox/xbm。

最大化按钮

max.xbm

默认值 系统内置

最大化按钮正常未被按下时

max_toggled.xbm

默认值 指定的 max.xbm ，否则使用系统内置图案

最大化按钮切换态图案

max_pressed.xbm

默认值 max.xbm, 内置图案

最大化按钮被按下时

max_disabled.xbm

默认值 max.xbm, 内置图案

最大化按钮不可点击时

max_hover.xbm

默认值 max.xbm, 内置图案

最大化按钮被鼠标指向时

max_toggled_pressed.xbm

默认值 max_toggled.xbm, max.xbm, 内置图案

最大化按钮切换被按下时

max_toggled_hover.xbm

默认值 max_toggled.xbm, max.xbm, 内置图案

最大化按钮切换被鼠标指向时

默认值 系统内置

最小化按钮被按下时

iconify_disabled.xbm

默认值 iconify.xbm, 内置图案

最小化按钮不可点击时

iconify_hover.xbm

默认值 iconify.xbm, 内置图案

最小化按钮被鼠标指向时

All-desktops button

desk.xbm

默认值 Internal default

All-desktops button in its default, unpressed state.

desk_toggled.xbm

默认值 If desk.xbm is present, it uses that. If not, it has a separate internal default

All-desktops button when it is in toggled state.

desk_pressed.xbm

默认值 desk.xbm, or its internal default

All-desktops button when pressed.

desk_disabled.xbm

默认值 desk.xbm, or its internal default

All-desktops button when disabled.

desk_hover.xbm

默认值 desk.xbm, or its internal default

All-desktops button when mouse is over it.

desk_toggled_pressed.xbm

默认值 desk_toggled.xbm, or desk.xbm, or its internal default

All-desktops button when pressed, in toggled state.

desk_toggled_hover.xbm

默认值 desk_toggled.xbm, or desk.xbm, or its internal default

All-desktops button when mouse is over it, in toggled state.
